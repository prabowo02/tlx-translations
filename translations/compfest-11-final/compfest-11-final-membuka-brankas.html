Description

Pak Chanek has a safe with $N$ letters as its password. $A_i$ shows the actual password of the safe at position $i$. To open the safe, we need to make the input password to be equal to the actual password. At first, the $i$-th character in the input is $B_i$.

The unique thing about the safe is that, to equalize the passwords, Pak Chanek can do two operations. The first operation is Pak Chanek changing the character at a certain position. This operation takes 1 second. The second operation is for Pak Chanek to pick two integers $i$ and $j$ such that $A_i = B_j$ and $A_j = B_i$. Then, Pak Chanek will swap the positions of the two characters in $B$ around. This operation also takes $1$ second.

Now Pak Chanek wonders, what is the minimum time for him to open the safe?

Input Format

The first line consists of an integer $T$ — the number of test cases. Each test case will be stated as follows.

The first line contains $N$ integers — the number of characters in the password.

The second line contains $N$ characters $A_i$ — the characters in the actual password.

The second line contains $N$ characters $B_i$ — the characters in the input password.

Output Format

For each test case, output an integer stating the minimum time required for Pak Chanek to open the safe (in seconds).

Sample Input

1
5
acbab
cccba

Sample Output

3

Explanation

Perform operation type 1 to position $1$, changing the character to a. The input password is now accba.

Perform operation type 1 to position $3$, changing the character to b. The input password is now acbba.

Perform operation type 2 with positions $4$ and $5$, swapping the two characters. The input password is now acbab.

Constraints

  • $1 \le T \le 10$
  • $1 \le N \le 100\,000$
  • The passowrd only consists of the characters 'a' - 'z'
